# Report Generator Skill

Transform content into clean, professional PDF reports. The output should look like it was designed by a human in Notion or a modern document tool — never like raw LLM output.

## When to Use

✅ **USE this skill when:**
- "Make this a PDF"
- "Generate a report from this"
- "Export this as a document"
- "Create a guide/handbook/reference"
- "Put this in a shareable format"
- User wants a polished deliverable from any analysis or conversation

❌ **DON'T use this skill when:**
- User just wants a text answer (don't generate PDFs unsolicited)
- Content is too short for a document (< 1 paragraph)

## Design Philosophy

The goal is **Notion-style output**: clean, minimal, professional, readable. The PDF should look like something a designer made, not something an AI generated.

### Core Principles

1. **Whitespace is your friend.** Generous margins, line spacing, and padding between sections. Let the content breathe.
2. **Typography matters.** Use a clean sans-serif font family (Noto Sans for Latin, Noto Sans KR for Korean). Consistent hierarchy: clear H1, H2, H3 with distinct sizes and weights.
3. **Color is minimal.** Light theme. No gradients, no heavy backgrounds. Use color sparingly for accents — a single accent color for headers, links, and highlights.
4. **No decorative noise.** No watermarks, no clip art, no stock icons, no "generated by AI" footers, no ornamental borders. Clean and functional.
5. **Content-first layout.** The reader's eye should go straight to the content. Navigation aids (table of contents, section headers, page numbers) support reading — they don't compete with it.

## Technical Implementation

### Method: HTML → PDF via Playwright

Generate an HTML file with embedded CSS, then convert to PDF using Playwright's Chromium.

```bash
# Install playwright if needed
npx playwright install chromium

# Convert HTML to PDF
node -e "
const { chromium } = require('playwright');
(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage();
  await page.goto('file:///absolute/path/to/report.html', { waitUntil: 'networkidle' });
  await page.pdf({
    path: 'output.pdf',
    format: 'A4',
    margin: { top: '20mm', right: '20mm', bottom: '20mm', left: '20mm' },
    printBackground: true
  });
  await browser.close();
})();
"
```

### Fallback: If Playwright is unavailable
Use `wkhtmltopdf` or `weasyprint` if installed. If no PDF tool is available, deliver the HTML file directly — it will still look professional.

### Font Loading

For Korean + English mixed content, include both font families:

```html
<style>
  @font-face {
    font-family: 'Noto Sans';
    src: url('/path/to/NotoSans-Regular.ttf') format('truetype');
    font-weight: 400;
  }
  @font-face {
    font-family: 'Noto Sans';
    src: url('/path/to/NotoSans-Bold.ttf') format('truetype');
    font-weight: 700;
  }
  @font-face {
    font-family: 'Noto Sans KR';
    src: url('/path/to/NotoSansKR-Regular.ttf') format('truetype');
    font-weight: 400;
  }
  body {
    font-family: 'Noto Sans', 'Noto Sans KR', -apple-system, BlinkMacSystemFont, sans-serif;
  }
</style>
```

Check for fonts in common locations:
- `/workspace/.fonts/`
- `/usr/share/fonts/`
- System font directories

If custom fonts are unavailable, fall back to system sans-serif. The design should still look clean.

## CSS Design System

Reference the full CSS in `references/base-styles.css`. Key design tokens:

### Colors
- **Background:** `#FFFFFF`
- **Text primary:** `#1A1A1A`
- **Text secondary:** `#6B7280`
- **Accent:** `#2563EB` (blue — for headers, links, highlights)
- **Accent light:** `#EFF6FF` (light blue — for callout backgrounds)
- **Border:** `#E5E7EB`
- **Code background:** `#F9FAFB`
- **Success:** `#059669`
- **Warning:** `#D97706`
- **Danger:** `#DC2626`

### Typography Scale
- **H1:** 28px, weight 700, color accent
- **H2:** 22px, weight 700, color primary, top border `1px solid #E5E7EB`
- **H3:** 18px, weight 600, color primary
- **Body:** 15px, weight 400, line-height 1.7, color primary
- **Small/caption:** 13px, weight 400, color secondary
- **Code:** 14px, monospace (Fira Code, Source Code Pro, or system monospace)

### Spacing
- **Section gap:** 48px
- **Paragraph gap:** 16px
- **Container max-width:** 720px (centered)
- **Page margins (PDF):** 20mm all sides

## Component Library

### Callout Box
For important notes, warnings, key takeaways:
```html
<div class="callout callout-info">
  <div class="callout-icon">💡</div>
  <div class="callout-content">
    <strong>Key Takeaway</strong>
    <p>Content here...</p>
  </div>
</div>
```

Variants: `callout-info` (blue), `callout-warning` (amber), `callout-danger` (red), `callout-success` (green)

### Data Table
Clean, minimal tables with alternating row colors:
```html
<table class="data-table">
  <thead><tr><th>Column</th><th>Column</th></tr></thead>
  <tbody><tr><td>Data</td><td>Data</td></tr></tbody>
</table>
```

For Scripture, expert quotes, or citations:
```html

  <p>"Quote text here."</p>

</blockquote>
```

### Bullet List (Styled)
```html

</ul>
```

### Tag/Badge
For categories, labels, status:

### Divider


## Content Adaptation Rules

### From Conversation/Chat → Report
When converting a chat conversation into a report:
1. **Remove conversational elements** — no "Great question!", no "Let me think about this", no back-and-forth artifacts
2. **Restructure for linear reading** — conversations jump around; reports flow logically
3. **Add section headers** — break content into scannable sections
4. **Add a summary/key takeaways section** at the top or bottom
5. **Add sources/references section** if citations were given
6. **Add table of contents** if the report has 4+ major sections

### From Bullet Points → Report
1. **Group related bullets under headers**
2. **Expand terse points** into readable sentences where needed
3. **Add introductory context** — the report should stand alone without the original conversation

### From Analysis → Report
1. **Lead with the conclusion/recommendation** (executive summary)
2. **Then show the evidence/reasoning**
3. **End with next steps or action items** if applicable

## Report Types

Adapt the template based on what kind of report it is:

### Research Report
- Title, date, author/source
- Executive summary (2-3 sentences)
- Table of contents
- Sections with findings
- Sources/references at the end

### Guide/Handbook
- Title, who it's for
- Quick reference section at top (cheat sheet, key facts)
- Progressive sections (basic → advanced)
- Practical tips, how-to steps
- Resources for further learning

### Comparison/Analysis
- Title, what's being compared
- Summary table or matrix at top
- Detailed breakdown per item
- Final recommendation with reasoning

### Theological/Exegetical Report
- Title, passage/topic
- Key texts listed upfront
- Position analysis sections
- Commentarial tradition
- Conclusion with practical application
- Scripture references appendix

## Quality Checklist

Before delivering the PDF, verify:
- [ ] Title is clear and descriptive
- [ ] Date is included
- [ ] Typography is consistent (no mixed sizes or weights)
- [ ] Korean text renders correctly (if applicable)
- [ ] Links are functional (or clearly formatted as URLs)
- [ ] No orphaned headers (header at bottom of page with content on next page)
- [ ] Callout boxes and tables don't break across pages awkwardly
- [ ] Footer/page numbers are present for reports > 3 pages
- [ ] The document looks like a human designed it, not like LLM output
- [ ] No "Generated by AI" artifacts, no markdown syntax visible in output

## File Delivery

After generating the PDF:
1. Confirm file size and page count
2. If in a sandboxed environment, use a temporary file sharing service (e.g., tmpfiles.org) to make it downloadable
3. Provide the download link to the user

```bash
# Upload to tmpfiles.org (files expire after a set period)
curl -F "file=@output.pdf" https://tmpfiles.org/api/v1/upload
```
